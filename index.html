Projetos que eu Participei


    Vert Analytics - Simulador de Políticas Tributárias da Secretaria da Fazenda do Estado do Rio Grande do Sul.
    Objetivo automatização das buscas das empresas e seus DEFIS, anteriormente realizadas manualmente no EXCEL.

  • Ocupação Estagiário Cientista de dados

    • Primeira fase

  • Toda a massa de dados foi migrada para o PostgreSQL.
  • Na linguagem SQL, foram efetuados todos os tratamentos dos dados, como valores ausentes, duplicados e fora de contexto, em colaboração com a equipe da SEFAZ, a qual auxiliou na identificação de anomalias e na confirmação da informação correta
  • Foram elaboradas todas as documentações necessárias em conjunto com a equipe da SEFAZ.

    • Segunda fase

  • Com os dados tratados, foram carregados no SAS Guide, uma ferramenta que permite extrair dados do banco de dados e agrupá-los por empresa, DEFIS e tipo de produto conforme a demanda do cliente.
  • Após diversos testes e ajustes na ferramenta e na base de dados dentro do SAS Guide, a equipe da SEFAZ começou a realizar as seleções de DEFIS por meio da ferramenta, validando assim o trabalho.
  • Após a validação, foi elaborada uma documentação em conjunto com a SEFAZ, detalhando o passo a passo de como utilizar a ferramenta, desde o carregamento dos dados até a aplicação de filtros.

    • Terceira parte

  • O SAS Guide foi conectado ao SAS Visual Analytics
  • Após a conclusão das operações no SAS Guide, os dados filtrados são transferidos para o VA (Visual Analytics), um ambiente utilizado para extrair dados e criar dashboards para apresentação.
  • Após diversos testes com os dados e os dashboards, foi elaborada uma documentação detalhada de como utilizar os dados no VA.

    • Tecnologias

    SQL/PosgreSQL, SAS, SAS Visual Analytics, SAS Guide


    Vert Analytics - Classificador de risco das DITIS Fazenda do Estado do Rio Grande do Sul.
    O objetivo deste projeto consiste em analisar, preparar e modelar os dados das Declarações de Informações sobre Movimentações Financeiras (DITS), visando oferecer à equipe da SEFAZ ferramentas e informações para uma tomada de decisão mais precisa e eficaz em relação aos riscos associados a essas movimentações anteriormente realizadas manualmente no EXCEL.

  • Ocupação Estagiário Cientista de dados

    • Primeira fase

  • Realização de reunião com a equipe da SEFAZ para aprofundar o entendimento sobre as Declarações de Informações sobre Movimentações Financeiras (DITS).
  • Compreensão dos cálculos necessários e dos procedimentos a serem seguidos.
  • Posteriormente, procede-se à análise das bases de dados no PostgreSQL.

    • Segunda fase

  • Após o completo entendimento dos dados, desenvolvem-se as fórmulas a serem aplicadas para a preparação dos dados para a próxima etapa, utilizando SQL no PostgreSQL.
  • Após o tratamento dos dados e a criação de novas bases a partir dos dados existentes, assim como a formulação de novas variáveis a partir dos cálculos realizados, as bases são entregues à equipe da SEFAZ para validação.
  • Após a validação, elabora-se um dicionário de dados e toda a documentação pertinente, descrevendo os processos realizados, as bases de cálculos e explicando as novas variáveis.

    • Terceira parte

  • Com a conclusão da base de dados, procede-se à criação dos modelos utilizando Python, com atualização em tempo real conforme novos dados são inseridos na base.
  • O primeiro modelo consiste em um algoritmo de clustering para categorizar as DITS em risco e não risco.
  • O segundo modelo é um algoritmo de classificação multivariada para classificar o risco das DITS, ordenando-as do maior para o menor.
  • Após a validação, todos os modelos são documentados em conjunto com a equipe da SEFAZ.
  • Após a obtenção desses resultados, é elaborado um relatório enviado ao analista da SEFAZ contendo as informações necessárias para que ele possa tomar a decisão final, em consonância com o trabalho realizado pela SEFAZ.

    • Tecnologias

    SQL/PosgreSQL,Python, Machine Learning
